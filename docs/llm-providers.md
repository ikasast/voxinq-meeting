# LLM providers

Minutes are generated by an LLM you choose in **Settings → LLM**. Transcription (Whisper) is
always local; only the minutes step uses this provider.

## Ollama (default, local)

- Easiest: model management built in, auto GPU/CPU.
- Best for the single-box setup. Default model `qwen2.5:7b-instruct` fits 8 GB VRAM.
- Set `Base URL` (`http://localhost:11434`) and `Model` in Settings.

## OpenAI-compatible (vLLM / LM Studio / OpenAI)

Anything that serves the OpenAI `/chat/completions` API. The **API key is optional** — local
servers need none.

| Server | Base URL | Key |
| --- | --- | --- |
| **LM Studio** | `http://localhost:1234/v1` | none |
| **vLLM** | `http://<host>:8000/v1` | none (unless you set one) |
| **OpenAI** | `https://api.openai.com/v1` | required |

- **LM Studio** — GUI desktop app; easy to run local models on a workstation.
- **vLLM** — high throughput and large context; heavier to set up and needs more VRAM. Great
  on a bigger/external GPU.

## Anthropic

Set `Model` (e.g. `claude-sonnet-4-6`) and `API key` in Settings.

## Choosing per run

The minutes **Regenerate** panel (sliders icon) lets you pick a **detail level** and
**provider** for a single run without changing saved settings — e.g. regenerate one meeting
with a bigger cloud model.

## Using an external GPU (e.g. Runpod)

Good when you want a larger model or larger context than the local box allows.

1. On the rented GPU, serve an OpenAI-compatible endpoint (e.g. **vLLM**) or run **Ollama**.
2. **Secure it** — do not expose it openly. Put the pod on Tailscale, or front it with an
   authenticated proxy.
3. In **Settings → LLM**, pick "OpenAI-compatible", set the Base URL (`https://<pod>/v1`),
   model name, and a key if required. (For Ollama on the pod, use the Ollama provider + its URL.)

Notes:
- The OpenAI-compatible provider has a large context budget, so long meetings run in a single
  pass (no chunked summarization needed).
- Runpod bills by uptime — start the pod only when generating, or keep it for a batch.

## Context length & long meetings

- Ollama's context is sized automatically from the input (up to ~24k tokens on 8 GB).
- If a transcript exceeds the budget, Voxinq **summarizes it in chunks then merges** — the
  latter half of a long meeting is never silently dropped. This makes multiple LLM passes, so
  it is slower locally; an external GPU with a big-context model avoids the chunking.
